Git en ligne de commande :Git Bash

Commandes de bases de GIT

  • pwd : print working directory affiche le Chemin du dossier où on se trouve
  • mkdir XX : Make directory Créer un dossier
  • cd XX/ : change directory Aller dans ce dossier. cd .. remonte d'1 dossier
  • git init : Crée un dépôt local vide dans le dossier
  • ls : list - Lister tous les dossiers et fichiers du dossier courant
  • Chemin total /d/Données/"Mes documents"/Dev/site/apprendre-a-coder
  • git ou git --help : aide

    • Configuration

  • git config --global user.name "prénom et nom" et git config --global user.email [email].
  • Pour vérifier : git config --global --list

    • Gestion des dépots dans un dossier local

  • git init : Crée un dépôt local vide dans le dossier
  • Termes : repository = dépots; enregistrement = commit
  • git add nomdufichier : ajouter les modifications à la zone d'index
  • git status : vérifier que les fichiers sont prets à être enregistrés
  • git reset nomdufichier : enlever le fichier de la zone d'index
  • git commit -m"nom des modifs" : enregistrer les modifications dans le dépôt local
  • git diff : voir les modifications qui ont été faites
  • git log : Historique des commits

    • Les branches

  • git branch mabranche : création d'une branche
  • git switch nomdelabrance : changement de branche
  • git switch -c nomdelanouvellebranche : création d'une nouvelle branche
  • git restore : annule les modifications dans le dossier local, et avec --staged permet de désindexer
  • git merge : rappatrie les modifications faites sur une branche sur une autre. Ex modif sur branche1, on se met sur master on merge et on a les modifs de banche1. Garde l'historique sur les branches.
      je suis dans ma branche1 -> git switch master
      je suis dans mon master je merge ma banche1 : git merge branche1
      régler les conflits danns le fichier code.txt et enregistrer ce fichier
      git status on voit que le fichier n'est pas indexé
      git add code.txt
      git commit -> éditeur de texte qui s'ouvre et qui dit que les conflits sont réglés et que l'on a fait un merge de la branche1 sur le master
  • git rebase : permet de mettre les modifs de 2 branches sur une même branche. Historique sur 1 seule branche.
      se mettre sur la branche1 à "rebase"
      git rebase master : on a des conflits -> fichier code.txt. Modifier le fichier pour régler les conflits et enregistrer
      git add code.txt quand conflits réglés
      git rebase --continue : éditer à nouveau le fichier code.txt pour régler les conflits
      git add code.txt
      git rebase --continue
      git checkout master
      git merge branche1
