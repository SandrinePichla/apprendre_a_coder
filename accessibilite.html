Accessibilité - Règles pour l'accessibilité des contenus web (Web Content Accessibility Guidelines, ou WCAG)

Directives : L'accessibilité: perciptible, opérationnel, compréhension, robustesse

  • perciptible
    • alternative textuelle pour tout contenu non textuel => alt images
    • audio= lecteurs d'écran
    • toucher = braille
  • opérationnel pour tous : souris, clavier, toucher, voix, boutons
  • compréhensible = lisible et prévisible
  • Robuste = lisible sur toute technologie d'assistance, et tout navigateur = compatible

Les erreurs

  • Mauvais contraste, inclure du texte dans un bloc image (impossible de grossir le texte)
  • Ne pas gérer le son : l'utilisateur utilise un lecteur d'écran, il doit pouvoir éteindre le son de la pub par ex
  • pensez à Accesibilité des éléments au clavier (lecteur d'écran)
  • Pensez au délai par exemple temps de connexion, pouvoir prolonger la session ...
  • faire clignoter un élement à une fréquence supérieure à 3 flashs par seconde. Le contenu qui clignote peut déclencher des crises d'épilepsie chez certaines personnes
  • Proposer un accès direct au contenu plutôt que de devoir naviguer dans tous les liens
  • balisage clair, titre unique par page, langue
  • Incohérence des élements : il faut que les élémnts qui se répètent se ressembles (ex barre de recherche...)
  • Penser à mettre de l'aide dans les formulaires : identifiant attendu, format ..., si erreur, donner le motif ..

SEO

SEO = référencement naturel (# payant)

Le SEO est un canal marketing, il y a donc des notions d'opportunité et de timing à prendre en compte avant de le lancer

Choisissez correctement les KPIs correspondant à vos objectifs. indicateurs-clés de performance=> servent à mesurer la performance des efforts SEO..

  • Visiteurs uniques (VU)
  • Nombre de pages vues par session
  • Taux de conversion sur vos formulaires d'inscription / de paiement
  • Taux de visiteurs récurrents ....

Vérifiez que vous avez le temps et les ressources nécessaires à la mise en place du projet.

Roadmap :

  • court audit préalable : voir si le site est déjà aux normes SEO ou si un audit plus important est à planifier
  • Recherchez les bons mots-clés (courte traine et longue traine)
  • Optimisation et création de contenu
  • Liens vers les potentiels partenaires
  • Surveillez l'évolution de vos actions SEO => reporting/li>

Audit rapide du site

  • Détectez les problèmes importants (vitesse de chargement, pb de crawling
  • Analysez rapidement le SEO On-Page : baliss metas, titres, images, cohérence et longueur de contenu (600 mots), contenu dupliqué...
  • Examinez les positions de vos mots-clés
  • Vérifiez la qualité des liens existants

référencement => dire que le site existe, positionnement => remonter le site dans les premières pages. Le SEO, Search Engine Optimization, (à l'opposé du SEA = Search Engine Advertising, consiste à acheter des mots clef pour être mieux positionné) consiste à améliorer son positionnement par des optimisations sur son site afin de le rendre plus pertinent. Il n'est pas possible de payer pour être mieux placé qu'un concurrent, tout le monde a, au départ, les mêmes chances.

  • google search coonsole est le seul outil d'analyse SEO nous permettant de savoir comment les internautes interagissent avec nos pages dans les résultats Google : affichages des pages, sur quels mots clés et pour combien de clics.
  • Google Analytics est une plate-forme qui collecte des données à partir de vos sites Web et applications afin de créer des rapports qui fournissent des insights sur votre activité
  • les spiders (robots) lisent le code html, ils vont parcourir les pages (crawl):
    • si le temps de chargement de chaque page est long, le robot en lira moins, et le référencement sera moins bon => améliorer les performance avant tout
    • s'il n'y a pas de lien sur la page = pas de lecture, s'il y a un menu nav il va savoir qu'il faut aller voir ici
    • il faut penser au référencement page par page
    • il faut mettre seulement des mots clés pertinents, pas trop.
    • utiliser la localisation => google My business, pages locales, citation (tripadvisor..), gérer les avis
  • Importance des balises h1, h2.. 1 seul h1, et bien hierarchiser
  • importnce de la balise title : visible sur l'onglet => pour s'y retrouver, et pour le référencement => 1 seule page avec le même titre
  • linkage : sur un lien toujours mettre un title, pour une image => toujours mettre un alt
  • meta
    • meta description très important => affichage dans la recherche
    • mots clés dans le contenu => mots pertinants
    • avoir une logique dans la hierachie des pages

il faut que le contenu du site soit logique, pertinant et bien balisé. Pour affiner, politique de mots clés sur URL, sur les contenus, sur balise title...Beaucoup d'experimentation, du temps pour tester et revenir en arrière, modifier...
