LA SYNTHAXE JSX

Balises sémantiques dans le fichier xxx.jsx

  • les élements écrits dans les balises dans le fichier .sjx forment un noeud react
  • 1 seul élément à la racine,par exemple une div, puis mettre les autres à l'interieur
  • par Ex : balise div, puis dedans h1, et dedans p...
  • ID : on peut mettre des id dans les balises comme en html
  • Attribut class se nomme className (camelcase sauf aria- et data-)
  • On peut utiliser l'élement "fragment" (ou balise vide, ouverte et fermée) à la place de div, toujours à la racine, ça permet d'avoir plusieurs éléments de premier niveau à l'interieur
  • toutes les balises doivent être fermée, par exemple img, ou input...Ou autofermantes

INTERPOLATION de variables JS

  • on crée une variable ex : const title
  • à l'interieur de la balise, on peut interpoler cette variable en l'appelant entre {}
  • exemple
  • on peut interpoler des variables dans les balises et dans les attributs des balises
  • Dans ces {}, on peut renvoyer n'importe quel noeud react, des chaines de caractères, des chiffres, null, undefined, mais pas d'objets => objet n'est pas un enfant react valide
  • on peut mettre des atrtibuts, type id, className, mais aussi attribut style (=css) mais il faut le noter sous forme d'objet. Attention toujours en camelCase
